# Changelog

## April 2026

- **Empty states, initial collapse, and map guide overlay** — Five first-run polish items plus one pre-existing bug fix:
  1. **All panels start collapsed on fresh page load.** New `setInitialCollapseState()` runs once after `initCollapse()` completes. Handles both drawer collapse mechanisms: `.card-body.collapsed` for `initCollapse`-managed cards, plus `style.display = 'none'` on the inline-toggle inner bodies (`#critBody`, `#pdcBody`, `#mbs010Body`, `#objectsPanelBody`) that have their own onclick handlers. The drawer opens as a compact list of section headers — users expand what they need. After initial collapse, the user has full manual control for the rest of the session (nothing auto-collapses or auto-expands).
  2. **Expand all on Load Assessment.** New `expandAllPanels()` reverses the collapse on both mechanisms. Called inside `loadAssessment()`'s success path (after the 800 ms render settle) so restored data is immediately visible.
  3. **Empty-state guidance messages inside key panels.** `.panel-empty-state` divs inserted in the drawer IIFE before the Receivers & criteria body, the Day situation source table (`#daySrcTable`), and the Predicted noise levels table (`#predTableDay`). They're hidden by default; the new `updateEmptyStates()` (called at the end of `render()`) adds `.show` when the relevant data is missing, and moves the underlying table off-screen via `position: absolute; left: -9999px` so the table stays in the DOM for Save/Load/Report but doesn't visually compete with the guidance message.
  4. **Map guide overlay.** `#map-guide-overlay` is inserted inside `#mapInnerWrapper` with `pointer-events: none` so map clicks pass through. Shows "Search for an address or click the map to place a noise source" when nothing is placed. `updateEmptyStates()` adds `.guide-hidden` as soon as any source or receiver exists, with a 0.4s opacity fade.
  5. **Contextual compliance strip messages.** Replaced the single `"Place source and receivers to see compliance"` fallback with four contextual states: `"Click the map to place a noise source and receivers"` (nothing placed), `"Source placed — now add receivers at sensitive locations"` (source only), `"Receivers placed — now add a noise source"` (receivers only), `"Configure noise sources to see predictions"` (both placed but no Lw data). Normal pred/crit/margin display takes over once all three ingredients exist.

  Also fixed a pre-existing crash: the drawer `]` keyboard shortcut handler used `e.target.matches(...)` unguarded — when a keydown event fired with `e.target === document` (e.g. after certain focus transitions), `document.matches` is undefined and the handler threw `TypeError: e.target.matches is not a function`. Now guarded with `e.target && e.target.matches && e.target.matches(...)`.

  No calculation, criteria, or Save/Load logic was touched; ISO/TR 17534-3 validation still passes.

- **Dropdown max-height fix for small viewports** — The Tools / Mapping / Modelling toolbar dropdowns used a static `max-height: calc(100vh - 120px)` that didn't account for where the dropdown actually starts on screen. The dropdown top is ~165px down from the viewport top (app header + toolbar row + padding), so on a 700px viewport the formula allowed `580px` but only `519px` was actually available below — the bottom ~45px of the menu got clipped below the viewport edge, and the last items (Show/hide objects, Show/hide boundary, Reset overrides, Clear All) were unreachable even with `overflow-y: auto`. Fix: `toggleMapPanel()` now measures `dropdown.getBoundingClientRect().top` at open time and sets `maxHeight = max(200, window.innerHeight - top - 16)` inline. This recalculates every time the dropdown is opened, so browser resizes automatically get correct max-height on next open. The static CSS `max-height: calc(100vh - 120px)` rule stays as a fallback for the brief moment before JS runs. Verified at simulated 700px viewport: Tools dropdown max-height clamps to 519px, content scrolls, last item "Clear All" is reachable. No calculation, state, or Save/Load logic touched; ISO/TR 17534-3 validation still passes.

- **Phase 3: Drawer resize handle with localStorage persistence** — Added a 6px drag handle (`#drawer-resize-handle`) on the left edge of the right-side drawer. Users can drag the handle to make the drawer wider (better for the 9-column criteria table and 10-band octave spectrum) or narrower (more map visible for source placement and noise mapping). Width is persisted to `localStorage` key `resonate_drawer_width` on drag end, restored on load, and clamped to min 360px / max 85% of viewport. The clamp runs on drag, on restore, and on window resize (so shrinking the browser never leaves the drawer wider than the viewport). Uses pointer events with `setPointerCapture` for smooth drag across the map area. Leaflet `invalidateSize()` fires on drag end and on re-clamp so the map contour layers rescale correctly. The handle is the first child of `#drawer-panel` (before toggle/header/content) and survives the PDF export round-trip untouched because `_restoreForPdfExport` / `_restoreDrawerLayout` only move `#drawer-content` children. No calculation or state logic was touched.
