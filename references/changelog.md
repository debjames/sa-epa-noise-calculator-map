- **ISO 9613-2 §7.4 ground-barrier interaction — sub-path Agr** — Implemented the §7.4 insertion-loss form of the combined barrier + ground term `AgrBar = max(Dz, Agr_subpath)`. Previously the calculator used `max(Dz, max(Agr, 0))` with the unobstructed-path Agr, which ignored the fact that when a barrier is present the ground effect should be recomputed along the source→barrier and barrier→receiver legs (the barrier top acts as a pseudo-receiver then pseudo-source). **New function** [`calcAgrBarrier(hS, hR, dp, G, barrierInfo)`](../shared-calc.js) in `shared-calc.js` calls `calcAgrPerBand` twice — once with `(hS, hBar, d1, {Gs, Gr: Gm, Gm})` for the source-side sub-path and once with `(hBar, hR, d2, {Gs: Gm, Gr, Gm})` for the receiver-side sub-path — and returns the per-band sum. **`calcISOatPoint` / `calcISOatPointDetailed`** gained an optional `barrierInfo = {d1, d2, hBar}` parameter (horizontal sub-path distances + barrier top height above ground). When provided the per-band loop uses `max(Ascreen, Agr_bar)` instead of `max(Ascreen, Agr)`. When absent the function falls back to the unobstructed-path Agr (backward compatible). The `Agr ≥ 0` clamp inside the barrier branch was removed: because `Ascreen > 0` inside the branch, `max(Ascreen, Agr)` already prevents negative Agr from reducing the total attenuation, so the clamp was redundant. **`getDominantBarrier`** now returns `d1` and `d2` directly in its result object (previously they were internal locals). **Call sites updated to thread `barrierInfo`**: `noise-worker.js` main loop (ISO Lmax, ISO Leq, and flat-spectrum-with-zones branches, including the over-top/gap energy combine paths — 6 call sites, constructed once as `_barrInfoW` per source-receiver pair); `index.html` line source loop (`bwLS`/`barrInfoLS`); area source loop (`bwAS`/`barrInfoAS`); building source propagation path (`barrInfoBs` in `_bsPropagateSub`); source-pin Leq path (`_barrInfoSP` threaded through the 6 `calcISO9613single` calls); source-pin Lmax path (`_barrInfoLmax` threaded through its 6 `calcISOatPoint` calls, replacing a dead `terrainPerBand` 10th-arg that was silently being dropped before the Deygout terrain refactor). `calcISO9613single` gained an 11th parameter `barrierInfo` that it passes straight to `calcISOatPoint` and `calcISOatPointDetailed`. **Test updates** — `iso17534.test.js` helper `computePerBandLA_barrier` now accepts `barrierInfo` and uses `calcAgrBarrier` when supplied; T08/T09/T11 tests pass hand-computed `{d1, d2, hBar}` from their Cartesian geometry.
